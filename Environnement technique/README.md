Cahier de charge environnement technique 
projet: architecture des contraintes et des qualités techniques 

CONTEXTE DU PROJET
     L'objectif consiste à traduire les besoins du client et les réalités du projet en un cadre technique claire et contraignant qui guidera toutes les décisions de développement 
ENVIRONNEMENT TECHNIQUE 
    C'est donc la description de l'écosystème dans lequel le logiciel doit vivre et être développé.
    1-Environnement cible( production)
    C'est l'espace dans lequel le logiciel sera utilisé par les utilisateurs finaux . Il comporte en outre:
    -Les systèmes d'exploitation (Windows, Linux.....)
    -Navigateur web(chrome Firefox....)
    -Matériel(serveur, smartphone...)
    -Environnement réseau ( intranet , accès internet require débit minimun)
2-Environnement de développement et de déploiement 
  -Langage de programmation, framework imposes)
  -Bases de donées (MYSQL, POSTGRESQL)
  -Outils de gestion de versions (github , gitlab)
  -Serveur d'application ( nodejs, apache Tomcat)
  -Plateforme de déploiement (cloud:AWS.azure.googlecloud)
